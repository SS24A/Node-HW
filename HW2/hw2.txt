Направете express апликација со следните рути:
- GET /users => низа од корисници
- GET /users/:id => корисникот со id-то од патеката
- POST /users => додава корисник во низата и го враќа ново додадениот корисник
- PUT /users/:id => го апдејтира корисникот со id-то од патеката
- DELETE /users/:id => го брише корисникот со id-то од патеката

  Секој корисник има по 2 props:
- email => уникатен за секој корисник
- password => низа од 6 до 10 карактери
пример:
{
  email: 'nenad@smx.com',
  password: 'password123'
}

id во контекст на рутите би било email полето, што значи при додавање на нов корисник (POST) треба да проверите дали во низата постои таков корисник, 
и доколку постои да вратите грешка со статус код 400 (Bad Request) и порака дека постои корисник со тој email.
Дополнително доколку се направи Request 
GET /users/nenad@smx.com
Треба да го врати корисникот со тој id (односно email)
{
  email: 'nenad@smx.com',
  password: 'password123'
}

Не додавајте нови полиља за id.
Користете express и nodemon во проектот соодветно инсталирани преку npm.
Сетирајте го express да препознава JSON преку app.use(express.json()).
